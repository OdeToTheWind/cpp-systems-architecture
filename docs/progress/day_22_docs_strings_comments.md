# Day 22 – Documentation, Strings vs Comments (Doxygen Style)

**Date:** April 04, 2026  
**Status:** Completed

## Goals
- Understand the difference between code comments and proper documentation
- Learn Doxygen-style documentation
- Practice writing self-documenting code + useful comments
- Generate documentation from code

## What I Learned / Key Takeaways

- Comments should explain **why**, not **what** (code should be self-documenting)
- Doxygen-style comments (`///` or `/** */`) can generate beautiful HTML/PDF docs
- Good documentation is part of professional code quality
- Over-commenting is as bad as under-commenting

**Rule of thumb**: If you need a comment to explain what the code does, the code is probably not clear enough.
