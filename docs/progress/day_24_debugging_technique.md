**Date:** April 06, 2026

## Goals
- Learn systematic debugging strategies in C++
- Use `std::cerr` for debugging output
- Understand common debugging tools and techniques
- Practice finding and fixing bugs through interactive examples

## What I Learned / Key Takeaways

- Print debugging (`std::cerr`) is the fastest way to start
- Use meaningful debug messages with variable values
- Divide and conquer: isolate the problem
- Common bugs: off-by-one, uninitialized variables, logic errors, null pointers
- Good debugging habit: reproduce the bug first, then fix it
